# vri-recommender

**Метод выбора вида разрешённого использования (ВРИ) территории проектирования
на основе наивного байесовского классификатора с ядерной оценкой плотности
по Сильверману.**

Магистерская диссертация, ИТМО, Институт дизайна и урбанистики, направление
07.04.04 «Градостроительство», 2026.

## Аннотация

Работа решает задачу автоматизированного подбора разрешённого использования
земельного участка по Классификатору № П/0412 на этапе предпроектного анализа.
Постановка — ранжирующая (top-k recommender), а не бинарная: для каждого
участка возвращается отсортированный список из укрупнённых классов ВРИ
с разложением апостериорной вероятности на два индекса:

- **P̃ — индекс потенциала** (вероятность класса при текущих характеристиках участка),
- **Ẽ — индекс эффекта** (типичность характеристик участка для класса).

Итоговый индекс пригодности агрегируется как геометрическое среднее:

$$S = \sqrt{\tilde{P} \cdot \tilde{E}}$$

Использование геометрического среднего отсекает варианты, у которых одна из
компонент близка к нулю, и выявляет ВРИ с одновременно высокими P̃ и Ẽ —
такие, для которых участок не только статистически вероятен, но и структурно
типичен.

Пилотная апробация — сельское поселение Манжерок, Майминский район,
Республика Алтай.

## Архитектура конвейера

```
Входные слои (geopandas + rasterio)
    ├── ЕГРН (196 695 участков с known ВРИ)
    ├── POI (точечные + полигональные)
    ├── УДС (дороги OSM, фильтр по уровням)
    ├── ЗОУИТ (10 типов охранных зон → бинарные флаги)
    └── DEM (растр высот → уклон в градусах)
                │
                ▼
    ┌───────────────────────────────────┐
    │  Признаки в метрической CRS       │
    │  • slope        (уклон, °)        │
    │  • land_cat     (категория земель)│
    │  • poi_density  (точек / км²)     │
    │  • road_dist    (м до УДС)        │
    │  • zouit_flags  (Z01…Z10)         │
    └─────────────────┬─────────────────┘
                      ▼
    ┌───────────────────────────────────┐
    │  Naive Bayes + Silverman KDE      │
    │  • P(x_j | C_k) = ∑KDE(x_j; h_k)  │
    │  • Лапласовское сглаживание       │
    │  • α = 0,74 (температурный        │
    │    параметр, откалиброван по CV)  │
    └─────────────────┬─────────────────┘
                      ▼
    ┌───────────────────────────────────┐
    │  P̃ × Ẽ → S = √(P·E)              │
    │  min-max нормализация в пределах  │
    │  одного участка                   │
    └─────────────────┬─────────────────┘
                      ▼
        Ранжированный список 13 ВРИ → results.json
```

## Метод (кратко)

1. **Признаковое описание.** Для каждого участка ЕГРН вычисляется вектор
   из пяти пространственных характеристик в метрической проекции
   (UTM 45N, EPSG:32645).
2. **Оценка плотностей.** Условные плотности `P(x_j | C_k)` строятся
   ядерной оценкой по Сильверману отдельно для каждой пары
   «признак × класс»; для категориальных признаков — нормализованная
   частотная гистограмма с лапласовским сглаживанием.
3. **Апостериор.** Совмещение по наивно-байесовскому правилу с равномерным
   приором (учёт только пространственной типизации, без априорного
   статистического перевеса в пользу массовых классов).
4. **Температурное сглаживание.** Параметр α = 0,74 откалиброван по
   минимуму CV-log-loss на сетке `[0,05; 1,5]` с шагом 0,05.
5. **Декомпозиция.** Апостериор раскладывается на P̃ (от участка к классу)
   и Ẽ (от класса к участку); финальный скор — геометрическое среднее
   `S = √(P · E)` после min-max нормализации внутри участка.
6. **Валидация.** Пространственная leave-one-out CV с пространственным
   буфером 800 м исключает утечку информации между близко расположенными
   участками.

Полное описание в `docs/method.md` и в файле `pipeline_v84.py` (929 строк
с подробными комментариями).

## Установка

```bash
git clone https://github.com/nikert/vri-recommender.git
cd vri-recommender
python -m venv .venv
source .venv/bin/activate              # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

Требуется Python ≥ 3.10. Зависимости — `geopandas`, `shapely`, `scipy`,
`rasterio`, `pyproj`, `matplotlib`, `numpy` (см. `requirements.txt`).
На Linux/macOS установка `geopandas` и `rasterio` через `pip` обычно работает
из коробки; на Windows может потребоваться conda
(`conda install -c conda-forge geopandas rasterio`).

## Запуск

После установки и подготовки папки с данными:

```bash
python pipeline_v84.py /path/to/data_dir/
```

Папка с данными должна содержать (имена файлов фиксированы):

| Файл | Обязательный | Описание |
| :--- | :---: | :--- |
| `egrn.geojson` или `egrn.gpkg` | ✓ | Участки ЕГРН с атрибутом ВРИ |
| `boundary_1.geojson` | ✓ | Хотя бы один пилотный участок |
| `boundary_2.geojson`, `boundary_3.geojson` | — | Дополнительные пилоты |
| `poi_point.geojson`, `poi_polygon.geojson` | — | Точки и полигоны POI |
| `roads.geojson` | — | Уличная сеть (OSM) |
| `zouit.geojson` | — | Зоны с особыми условиями использования |
| `dem.tif` | — | DEM для расчёта уклона (без него `slope = 0`) |

Результаты сохраняются в `data_dir/results_YYYY-MM-DD_HH-MM-SS/`:
`results.json`, набор SVG-визуализаций (приоры, KDE, точность CV,
матрица P × E) и PNG-картограмма пилотных участков.

Эти файлы превышают лимит GitHub в 100 МБ (для `egrn`) либо близки к
порогу предупреждения (для `zouit`), поэтому опубликованы как

`egrn_features.gpkg` → `egrn.gpkg`, `zouit.geojson` остаётся как есть.

Малые файлы (классификатор и пилотные boundary) лежат в `data/`
и версионируются вместе с кодом.
